## 3 — Sentences Tab Fix

**File:** `desktop/renderer/app.js`

**Problem:** The Sentences tab was empty for all words. `renderSentences()` was filtering with `words.filter(w => w.context)`, but `word.context` is always `''` because the popup captures the word text only, not surrounding sentence context.

**Fix:** Added `word.aiAnalysis?.exampleSentence` as a fallback source.

```js
// Before
const withContext = words.filter(w => w.context);

// After
const getSentence = w => w.context || w.aiAnalysis?.exampleSentence || '';
const withContext = words.filter(w => getSentence(w));
```

**Additional fix — DOM error:** `detail.insertBefore(aiLabel, sentence)` threw `NotFoundError` because `sentence` had not yet been appended to `detail`. Changed to `detail.appendChild(aiLabel)` followed by `detail.appendChild(sentence)`.

**"✦ AI example" pill:** Words using the AI example sentence get a small branded pill label to distinguish them from real context sentences.

## 4 — Game Mode Bug Fixes

**Files:** `game-engine.js`, `game-controller.js`, `game-modes.js`

### Bug 1 — streak_5 achievement unearnable (High)

**Problem:** `checkAchievements()` is called at session end when `sessionStats.streak` has already been reset to 0 by any wrong answer or skip. A player who builds a 10-word streak then gets one wrong never earns the badge.

**Fix:** Added `maxStreak` to `sessionStats`. Incremented whenever `streak` increases. Achievement condition changed to check `maxStreak`.

```js
// game-engine.js
{ id: 'streak_5', condition: (p, s) => s.maxStreak >= 5 }  // was s.streak

// game-controller.js — in processResult
sessionStats.streak += 1;
if (sessionStats.streak > sessionStats.maxStreak) sessionStats.maxStreak = sessionStats.streak;
```

### Bug 2 — Skip penalized identically to Wrong (Medium)

**Problem:** `skipWord()` called `processResult({ skipped: true })` which fell into the else branch and called `modeInstance?.onWrong?.()`. Skip in Survival mode would deduct 10 seconds and could end the session immediately — same penalty as a wrong answer. This contradicts the SRS design where skip is a soft action.

**Fix:** Added a dedicated `onSkip()` method to all three mode classes with gentler penalties. `processResult` now has a separate `skipped` branch.

```js
// game-modes.js
RaceMode.onSkip()    → progress −5%  (vs −10% for wrong)
SurvivalMode.onSkip() → timeLeft −5s, never triggers immediate end  (vs −10s + instant end)
MissionMode.onSkip()  → resets noHintStreak only, does not increment totalAnswered
```

### Bug 3 — calcXP called twice (Low)

**Problem:** `calcXP()` was called once to build the feedback HTML and again when persisting XP. Both calls were pure so values matched, but a future mutation between the two calls could silently diverge the displayed and credited XP.

**Fix:** Computed once, used twice.

```js
const xp = calcXP(word, result, selectedMode);
feedback.innerHTML = `... +${xp} XP`;
// ... later
if (xp > 0) await addXP(xp);
```

### Bug 6 — Unnecessary await on sync functions (Cosmetic)

`getWords()` and `getProfile()` in `storage-shim.js` are synchronous (they read from the in-memory cache / localStorage). They were `await`-ed in 8+ places in `game-controller.js` and `player.js`. Removed all unnecessary `await` calls.

## 5 — Race Mode: Word-Count Selector & Win Condition (Bug 4)

**Files:** `game-controller.js`, `game-modes.js`, `index.html`

**Problem:** Race mode had no win condition. Progress was clamped to 100% but never ended the game. A player answering everything correctly could play forever.

**Fix:** Added a word-count selector and a finite word queue.

**UI — Word picker (shown only when Race is selected):**
- Options: 5 words / 10 words (default) / 20 words / All words
- Displayed in a branded pill section below the mode cards in `screen-mode`

**Logic:**
- On `startSession()`, words are shuffled and sliced to the selected count → stored in `raceQueue[]`
- `nextRound()` pops from `raceQueue` instead of calling `selectNextWord()`
- When `raceQueue` is empty → `endSession()` (win)

```js
// startSession
const shuffled = [...words].sort(() => Math.random() - 0.5);
raceQueue = raceWordLimit === Infinity ? shuffled : shuffled.slice(0, raceWordLimit);

// nextRound (race mode)
if (!raceQueue.length) { endSession(); return; }
currentWord = raceQueue.pop();
```

The main process maintains `_wordCache[]` — a copy of the user's word list. The main renderer pushes it via IPC whenever the vocab changes.

```js
// app.js — helper
function pushWordCache() {
  if (window.electronAPI?.sendWordCache) window.electronAPI.sendWordCache(getWords());

// Called after: startup loadVocab, word save, word delete, sync
```

### Bug Fix — Logout White Screen (animationend Race Condition)

**Problem:** `showMainApp()` attached a `{ once: true }` `animationend` listener to `#login-page` to hide it after the `lp-exit` fade-out. When the user later logged out and `showLoginPage()` ran, it added `lp-enter` to the login page. The stale listener was still live and fired on the `lp-enter` animation's completion — running `lp.style.display = 'none'` again and instantly hiding the login page, leaving a pure white screen.

**Fix:** Introduced `_lpExitHandler` module-level variable to track the pending listener. `showMainApp()` stores the handler there; `showLoginPage()` calls `removeEventListener` on it before doing anything else. The `showLoginPage()` return path no longer adds any animation — it simply strips all animation classes and sets `display: flex` directly.

```js
let _lpExitHandler = null;

// showMainApp — stores the handler so it can be cancelled
_lpExitHandler = () => { lp.style.display='none'; lp.classList.remove('lp-exit'); _lpExitHandler=null; };
lp.addEventListener('animationend', _lpExitHandler, { once: true });

// showLoginPage — cancels it before touching the element
if (_lpExitHandler) { lp.removeEventListener('animationend', _lpExitHandler); _lpExitHandler=null; }
lp.classList.remove('lp-exit', 'lp-enter');
lp.style.display = 'flex';
```

## App Overview (as of 29 June)

Smart Translation (now Semantica) is an Electron desktop app for intermediate-to-advanced English learners (B2–C1 CEFR). It helps users build vocabulary by saving words encountered while browsing or reading, then provides AI-powered definitions, spaced repetition review, fill-in-the-blank practice, and gamified learning.

### Core Architecture

```
Smart Translation/
├── desktop/
│   ├── main.js               — Electron main process (OS, global shortcuts, IPC)
│   ├── preload.js            — Bridge between main and renderer (exposes electronAPI)
│   ├── preload-popup.js      — Bridge for the floating popup window
│   └── renderer/
│       ├── app.js            — Main dashboard UI (vocab, review, practice, game)
│       ├── popup.js          — Floating translate popup (triggered by Cmd+Shift+T)
│       ├── api.js            — Backend API client + in-memory word cache
│       ├── storage-shim.js   — Unified storage: getWords, saveWord, updateWord, deleteWord
│       ├── game-controller.js / game-engine.js — Gamification logic
│       ├── theme.js          — Dark/light mode + hue theming
│       ├── dom-utils.js      — escapeHtml and shared DOM helpers
│       ├── ielts-wordlist.json — Merged vocabulary lookup (8,828 entries)
│       └── index.html        — App shell, CSS, layout
└── server/
    └── src/
        ├── index.js          — Cloudflare Worker entry (Hono router)
        ├── auth.js           — OTP email auth (request code + verify)
        ├── analyze.js        — Claude Haiku 4.5 + DeepL word analysis
        ├── prompts.js        — System prompt with prompt caching
        ├── vocab.js          — Vocab/settings/profile CRUD endpoints
        ├── translate.js      — Translation endpoint
        └── rate-limit.js     — Per-user daily rate limits via KV
```

### Backend
- **Runtime:** Cloudflare Workers (Edge)
- **Router:** Hono
- **Database:** Cloudflare D1 (SQLite) — users, auth_codes tables
- **Cache:** Cloudflare KV — word analysis cache (7-day TTL), rate limits
- **AI:** Claude Haiku 4.5 via Anthropic API with prompt caching (≥1024 tokens)
- **Translation:** DeepL API (parallel with Claude)
- **Email:** Resend API (`onboarding@resend.dev` sender — sandbox at this point)

### Auth Flow
Email OTP — no passwords:
1. User enters email → POST `/v1/auth/request` → generates 6-digit code, hashes it (SHA-256), stores in D1, sends via Resend
2. User enters code → POST `/v1/auth/verify` → checks hash + expiry (10 min TTL), issues opaque bearer token
3. Token stored in `localStorage` (`cloud_auth_token`), sent as `Authorization: Bearer` on all subsequent requests

## Bug 1 — `ACCOUNT_ID` Placeholder in Cloudflare AI Gateway URL

**File:** `server/src/analyze.js`

The `ANTHROPIC_GATEWAY_URL` constant contained a literal `ACCOUNT_ID` placeholder never replaced with the real Cloudflare account ID. Requests to the AI Gateway failed with `403 forbidden`.

**Why Mac worked but Windows didn't:** Mac users' prior lookups were cached in KV (7-day TTL). Windows users had no cache, so every request hit the gateway and failed.

**Fix:**
```js
// Before
const ANTHROPIC_GATEWAY_URL = 'https://gateway.ai.cloudflare.com/v1/ACCOUNT_ID/smart-translation/anthropic/v1/messages';

// After
const ANTHROPIC_GATEWAY_URL = 'https://gateway.ai.cloudflare.com/v1/eeebef75914ebf0dddf7c498417a4e41/smart-translation/anthropic/v1/messages';
```

## Bug 3 — Popup Auth Token Returning null on Windows

**File:** `desktop/renderer/popup.js`

After fixing the gateway URL, Windows users still got `401 Unauthorized`. The popup gets its token via IPC from the main process. On Windows, the IPC call returned `null`.

**Beta.3 fix:** Added `localStorage` as a fallback:
```js
const ipcToken = await window.electronAPI.getToken();
const lsToken  = localStorage.getItem('cloud_auth_token');
const token    = ipcToken || lsToken;
```

**Beta.5 fix (final):** Token piggybacked on the `analyze-word` IPC message itself — eliminating the separate async `getToken()` call and the race condition it caused on Windows:
```js
// main.js
popupWindow.webContents.send('analyze-word', word, _authToken);

// preload-popup.js
onAnalyzeWord: (cb) => ipcRenderer.on('analyze-word', (_, word, token) => cb(word, token)),

// popup.js
window.electronAPI.onAnalyzeWord(async (word, token) => {
  const lsToken = localStorage.getItem('cloud_auth_token');
  _token = token || lsToken || null;
});
```
