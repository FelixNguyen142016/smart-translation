## 2 — Tauri v2 Migration (`semantica-tauri/`)

- Renderer copied byte-identical; all three Electron preloads replaced by ONE `src/tauri-bridge.js` exposing the same `window.electronAPI` surface via `invoke()`/`listen()`.
- Multi-arg `analyze-word` IPC → single object payload `{ word, token, savedData, wotd }` + `popup_ready` command handshake (fixes emit-before-listener race on first popup open).
- `main.js` → Rust `lib.rs`: `AppState` (Mutex: auth token, word cache, pending popup payload), commands `set_token`, `get_token`, `update_word_cache`, `search_word`, `popup_ready`, `show_word_of_day`, `show_dashboard`, `close_search`, `close_popup`.
- **Shortcut changed:** quick search is now `Cmd/Ctrl+Shift+D` (was the conflicting `Ctrl+Shift+F`). **`Cmd+Shift+T` translate shortcut removed** — replaced by the system tray icon.
- **System tray:** left-click drops the search bar under the icon; right-click menu = Open Dashboard / Quit.
- `-webkit-app-region: drag` → `data-tauri-drag-region` (nav, titlebar spacer, popup header, search wrap).
- Blur-dismiss for search + popup windows preserved.
- Placeholder icons generated; replace with `cargo tauri icon <1024px png>`.
- Run: `cargo tauri dev`. See `semantica-tauri/SETUP.md`.

## 3 — Worker CORS for Tauri

`tauri dev` serves the frontend from `http://127.0.0.1:1430` (not `tauri://localhost`), which the Worker blocked → "Load failed" on login. CORS now allows: `tauri://localhost` (prod macOS/Linux), `http://tauri.localhost` (prod Windows — http, not https), literal `"null"` origin (WKWebView edge case), and `http://(localhost|127.0.0.1):<any port>` (dev). **Deployed.**

## 8 — Server Pipeline v2 (`prompts.js` + `analyze.js`)

**Claude's job shrank to English-only:** definition, partOfSpeech, exampleSentence, tags, ieltsTopics (~110 output tokens vs ~300; `max_tokens` 1024→400). System prompt no longer contains the target language → **one Anthropic prompt-cache entry for all users** (still ≥1024 tokens).

**Phase 1 (parallel):** Claude · Google TTS · Datamuse synonyms (`rel_syn`, free) · IPA from KV.
**Phase 2:** ONE DeepL call translating `[word, definition, example]` → `translation`, `definitionTranslated`, `exampleSentenceTranslated`.
**Fallback:** if DeepL fails/rejects VI (next-gen-model language — some plans reject it), a tiny Claude translation call covers all three strings, so responses are never English-only.

Response shape unchanged — no renderer changes needed. Expected cache-miss latency ~1.3–1.5s (was ~3s); KV hits instant; Oxford-list words paint at 0ms client-side.

**IPA dataset:** `en_UK.txt` (65,119 words incl. inflections) → `server/ipa-kv/ipa-chunk-0[1-7].json`. Upload once:

```bash
cd server
for f in ipa-kv/ipa-chunk-*.json; do
  npx wrangler kv bulk put "$f" --namespace-id=132f13f0b38a4d4186430b30a90024cc
done
```

## 12 — Popup Error Fixes (end of session)

**Symptom:** some words (e.g. "autostart") showed "Could not fetch definition — The string did not match the expected pattern"; other words showed English-only.

**Analysis:** dictionary words mask API failures via the offline fallback; non-dictionary words expose them. Two causes addressed:
1. English-only = Phase-2 DeepL returning null for VI (plan/model support) → **Claude translation fallback added** (§8).
2. The pattern error is WebKit's `fetch` rejection for a malformed header → token now sanitized (trimmed, ASCII-validated) before building `Authorization`; `TypeError` failures show a human message ("Could not reach the Semantica server…"); dev builds auto-open popup devtools (`#[cfg(debug_assertions)]`) so the true console error is visible if it recurs.
