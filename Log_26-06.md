## App Overview

Smart Translation is an Electron desktop app for intermediate-to-advanced English learners (B2–C1 CEFR). It helps users build vocabulary by saving words encountered while browsing or reading, then provides AI-powered definitions, spaced repetition review, fill-in-the-blank practice, and gamified learning.

### Core Architecture

```
Smart Translation/
├── desktop/
│   ├── main.js               — Electron main process (OS, global shortcuts, IPC)
│   ├── preload.js            — Bridge between main and renderer (exposes electronAPI)
│   ├── preload-popup.js      — Bridge for the floating popup window
│   └── renderer/
│       ├── app.js            — Main dashboard UI (vocab, review, practice, game)
│       ├── popup.js          — Floating translate popup (triggered by Cmd+Shift+T)
│       ├── api.js            — Backend API client + in-memory word cache
│       ├── storage-shim.js   — Unified storage: getWords, saveWord, updateWord, deleteWord
│       ├── game-controller.js / game-engine.js — Gamification logic
│       ├── theme.js          — Dark/light mode + hue theming
│       ├── dom-utils.js      — escapeHtml and shared DOM helpers
│       ├── ielts-wordlist.json — Merged vocabulary lookup (8,828 entries)
│       └── index.html        — App shell, CSS, layout
└── server/
    └── src/
        ├── index.js          — Cloudflare Worker entry (Hono router)
        ├── auth.js           — OTP email auth (request code + verify)
        ├── analyze.js        — Claude Haiku 4.5 + DeepL word analysis
        ├── prompts.js        — System prompt with prompt caching
        ├── vocab.js          — Vocab/settings/profile CRUD endpoints
        ├── translate.js      — Translation endpoint
        └── rate-limit.js     — Per-user daily rate limits via KV
```

### Backend (Cloudflare Worker)
- **URL:** `https://smart-translation-api.fukumakino613.workers.dev`
- **Runtime:** Cloudflare Workers (Edge, no Node.js)
- **Router:** Hono
- **Database:** Cloudflare D1 (SQLite) — users, auth_codes tables
- **Cache:** Cloudflare KV — word analysis cache (7-day TTL), rate limits
- **AI:** Claude Haiku 4.5 via Anthropic API with prompt caching (≥1024 tokens)
- **Translation:** DeepL API (parallel with Claude, replaces Claude's translation field)
- **Email:** Resend API (`onboarding@resend.dev` sender for sandbox/dev)

### Auth Flow
Email OTP — no passwords:
1. User enters email → POST `/v1/auth/request` → generates 6-digit code, hashes it (SHA-256), stores in D1, sends via Resend
2. User enters code → POST `/v1/auth/verify` → checks hash + expiry (10 min TTL), issues opaque bearer token
3. Token stored in `localStorage` (`cloud_auth_token`), sent as `Authorization: Bearer` on all subsequent requests

## Features Implemented

### 1. Word Save — Cmd+Shift+T Popup
- Global shortcut triggers a floating Electron window
- Copies selected text automatically, calls `/v1/analyze`
- Shows definition, translation, example sentence
- Save button prepends word to top of vocab list (newest-first)
- Fixed: words were being appended to bottom (`push` → `unshift`)

### 2. AI Word Analysis
- **Endpoint:** POST `/v1/analyze`
- **Model:** Claude Haiku 4.5 with Anthropic prompt caching
- **Output fields:** `translation`, `definitionTranslated`, `definition`, `partOfSpeech`, `pronunciation` (IPA), `synonyms`, `exampleSentence`, `exampleSentenceTranslated`, `tags`, `ieltsTopics`
- DeepL runs in parallel for higher-quality translations
- Results cached in KV for 7 days

### 3. IELTS Topic Tagging
- AI returns `ieltsTopics` array (1–2 labels per word) from a fixed list of 25 IELTS exam topics
- Topics rendered as indigo chips in the definition panel
- Prompt rule added to `server/src/prompts.js` (rule 14)
- Zero extra cost — just additional JSON fields in the existing analyze call

### 4. Vocabulary Table
- Columns: Word | Context & Meaning | Tags | Actions
- **Word column:** gradient word name, IPA pronunciation, Play button (Web Speech API), Band pill (colour-coded by level)
- **Band pill colours:** Band 2–3 = slate, Band 5 = teal, Band 6 = blue, Band 7 = purple, Band 8 = amber, Band 9 = red
- **Context & Meaning column:** context sentence, YouTube jump-to-moment link (if from YouTube), AI definition card
- **Tags column:** AI-generated tags + user-created tags with × remove button, `+ tag` add button

### 5. Filter by Tag (Tag Cloud)
Restructured into 3 rows:
1. **Band filter pills** — coloured, show word count per band, clickable to filter
2. **IELTS topic chips** — indigo, show word count per topic, clickable to filter
3. **Auto-generated tags** — standard tags, collapsed to 2 lines with "See more ↓" expand button

Filter logic: AND across groups (band + topic + tag must all match if multiple selected). Active filter bar shows what's selected with word count. "Review these words" button passes filtered set to review session.

### 6. Spaced Repetition System (SRS)
SM-2 algorithm implementation in `app.js`:

```js
// grade: 0=Hard, 1=Good, 2=Easy
function srsUpdate(word, grade) { ... }
function srsStatus(word) { ... } // → 'new' | 'due' | 'soon' | 'future'
function formatDue(srs) { ... }  // → { label, color }
```

**SRS strip** shown at top of Review mode:
- Counts: Overdue | Due today | Upcoming | New
- "Due (N)" button — shows only overdue + due today, sorted by urgency
- "All (N)" button — shows all words sorted by SRS priority

**Card back face** shows next-due chip (🕐 Due now / Tomorrow / In N days) and review count.

**Button handlers** save `srs` field via `updateWord()`:
- Hard → interval resets to 1, easeFactor −0.2, word re-queued (max 2×)
- Good → interval ×1.5, easeFactor unchanged
- Easy → interval × easeFactor, easeFactor +0.1, word marked 'known'

### 7. Fill-in-the-Blank Practice Tab
New "Practice" nav tab with:
- Blanks the target word (and inflected forms) in its example sentence using regex
- User types the answer into an input field
- Hint button reveals first 2 characters
- Toggle to show/hide full sentence
- Answer checking: green ✓ correct / red ✗ wrong feedback
- End screen: score %, green "Correct" word pills, red "Needs work" word pills
- Restart button resets the queue

### 8. Review Mode — Restart Fix
Fixed broken restart button: `onclick = startReviewSession` was passing the click Event object as the `prefiltered` argument (truthy), causing `_startQueue(event, 'sorted')` to spread an empty array and immediately hit "Session Complete".

Fix: `onclick = () => startReviewSession()`

### 9. Vocab List UX
- **"Back to top" button** — appears at bottom of word list, scrolls `<main id="main-scroll">` (the actual scroll container with `overflow-y: auto`)
- **Login to test:** Use Account & Settings → Logout button (`#acct-logout`). Or in DevTools console: `localStorage.removeItem('cloud_auth_token')` then `location.reload()`

## Security Notes
- Never commit `.env`, API keys, or `wrangler.toml` secrets to git
- Token stored as SHA-256 hash in D1 — plaintext never persisted
- OTP codes stored as SHA-256 hash, deleted on first use (one-time)
- Rate limiting on `/v1/analyze` via KV counters (per userId per day)
